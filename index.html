Hi!

Projects

Music Analysis Website

A pet project of mine, the website started on a basic idea to try and understand my music a little better. As both a music and a data nerd, I loved being able to dissect how I listened - and wanted more detail than Spotify was providing.

Technologies
With HTML and CSS on the front end (with some more help from Bootstrap), my new knowledge on this project came from exploring different tools on the backend. Using Flask, I worked to understand HTTP and REST better as I implemented them in this website. This allowed me to plug the Spotify Python API into the backend, bringing me back to more familiar terrirtories of Python data analysis.
Challenges
This project brought me again to a central challenge of sustainable structural design. My goal was to create a website that would allow me to easily update or add new features without major difficulties. From the file structure to the layout of the website itself, I wanted to create something that would be user and developer friendly. In particular, updating documentation and more carefully following basic software design principles have become a primary focus for this project.
Future
As I grow the project, one of my goals is to add more customizable display options for the data. I also will consider tradeoffs between storing more data and frequent HTTP requests as I weigh the possibility of caching some user data in a database.

Action Lab Website

My first serious endeavour into web design, this project was guided and designed by the Action Lab at Northeastern University. In order to increase the ease with which data could be collected, a series of "prediction" games would be moved online. The website therefore had requirements to align with these needs: user data needed to be accurately and securely recorded, the data had to be accessible for easy analysis, and the games needed to be usable from any average home computer. As the project continued, there were of course new guidelines that were realized and standards to be met, but these requirements drove my initial designs.

Technologies
To make the website, I used HTML and CSS for the front end, making templates with Jinja2 and programming the games with Javascript. The backend was Python using a Google Web API to host my website and connect to a database. The security on the website was limited to self research on creating cookies and encrypting data using a python hashing library. While there are plenty of existing tools that would have made my job much easier (and the result more sustainable), my knowledge and background was limited and I hoped to keep the project under control by limiting my use of unfamiliar languages or libraries.
Challenges
With little techinical guidance, the challenges of this project were mostly in the longevity of its design and the makeshit solutions to permanent problems. My unfamiliarity with Javascript is evident in the poorly patched code and the inexplicable mess of variables. To address similar issues in the future, I will work on focusing on design from the very start of a project - anticipating future needs as well as their growth. In addition, I aim to research available technologies and principles before starting to code. As I continue to work, I've also begun to recognize the need for a mentor and will continue to seek guidance from a number of different technical individuals as I embark on different projects.
Future
I parted with this project for good as I began to work full time for Cisco in the summer of 2018. I had worked on the website off and on for about two years and was excited to start from scrath on new projects or my own. I have passed on my work, and the project will continue to be developed with a new grant the Action Lab received for a research proposal using the data collected to be collected with the games.
